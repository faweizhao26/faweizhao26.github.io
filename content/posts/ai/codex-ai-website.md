这几年，我逐渐养成了阅读电子书的习惯，也陆续收藏了不少电子书资源网站。

很早之前，我就产生过一个想法：如果能把这些分散的网站集中到一个平台上，让更多有需要的人方便访问，那该多好。

但现实是——我不是技术出身，对建站几乎一窍不通。虽然在网上查过一些教程和方案，但对我来说依然有门槛，于是这个想法被我搁置了好几年。

直到最近，AI 的发展让我重新燃起了这个念头。

---

## AI，让普通人也能“写代码”

过去两年，AI 的进化速度可以说是跳跃式的。

现在，我们已经可以通过自然语言，让 AI 自动帮我们生成、修改和优化代码。这种方式在业内被称为 **Vibe Coding（氛围感编程）**：人只负责表达想法和方向，具体实现交给 AI。

这不仅极大提升了程序员的效率，也让像我这样的非技术人员，有机会把想法真正变成产品。

在此之前，我已经用 AI 给自己的开源项目官网加过页面，体验过“零基础开发”的可能性。这次，我决定更进一步：直接用 AI 从零做一个网站。

---

## 为什么选择 Codex？

目前市面上的 AI Coding 工具越来越多，比如 Claude Code、OpenCode 等。

最终，我选择了 OpenAI 推出的 Codex。

简单来说，Codex 就是一个 AI 编程助手，你可以把它当成“24 小时在线的程序员搭档”。

它能帮你：

- 生成网站代码
- 修改页面样式
- 修复 Bug
- 部署上线
- 分析项目结构

而且整个过程，基本都靠对话完成。

官网：https://openai.com/index/introducing-codex/

---

## 准备工作：安装和配置

Codex 的安装方式很多，官网都有说明。

我个人更喜欢聊天式操作，所以直接安装了官方客户端，没有折腾命令行。

使用 Codex 需要成本支持：

- 要么购买 ChatGPT 订阅
- 要么使用 OpenAI API Key

考虑到性价比，我选择了 API Key 方案，在闲鱼购买了一个月套餐进行测试。卖家一般都会附带详细教程，照着配置就能用。

> 提醒一句：第三方渠道存在一定风险，重要项目建议走官方渠道。

---

## 从一句话开始建站

准备完成后，就可以正式开始“写网站”了。

我的第一句话大概是：

> 我想做一个电子书资源导航网站，有分类、有卡片，可以点击跳转。

然后，我补充了：

- 想要的页面结构
- 参考网站链接
- 风格偏好

Codex 很快就给我生成了一个本地网页，我直接在浏览器里预览效果。

接下来，就是不断对话优化：

- 这个地方改成卡片布局
- 分类再清晰一点
- 字体换大一些
- 首页加搜索框

整个过程，几乎就像在跟一个真正的程序员协作。

我做的网站结构比较简单，本质是一个目录导航站，没有复杂交互，所以整体开发时间不到两个小时。

---

## 部署上线，让别人也能访问

写完代码后，网站还只能在我自己电脑上访问。

要公开访问，就必须部署到线上平台，比如：

- GitHub Pages
- Netlify
- Vercel

因为我以前用过 GitHub Pages 搭博客，所以这次依然选择它。

不熟悉部署流程也没关系，直接让 Codex 帮你操作，授权后它可以自动完成提交和发布。

---

## 买个域名，让网站更“正式”

部署完成后，网站已经能访问了，但默认链接又长又难记，对 SEO 也不友好。

于是我又买了一个独立域名，并做了域名解析和 CDN 加速。

域名是在阿里云买的，一年几十块，成本很低。

至此，网站算是完整上线了。

## 使用过程中的几点经验

结合这次实践，我总结了几点体会：

### 1️⃣ 第三方 API 有风险

闲鱼方案便宜，但稳定性一般，重要项目谨慎使用。

### 2️⃣ 不会就问 AI

遇到问题，直接丢给 Codex，大多数都能解决。

### 3️⃣ 表达越清楚，效果越好

需求说得越细，AI 给的结果越接近预期。

---

## 写在最后

以前我一直觉得：做网站、写程序，是“技术人员的专属能力”。

但现在，AI 把这道门槛彻底打平了。

借助 Codex，我不仅完成了这个电子书导航站，还顺手重构了自己以前的博客样式。

很多过去需要几天甚至几周的事情，现在几个小时就能搞定。

对普通人来说，这可能是这个时代最重要的红利之一。

感谢 AI，让想法第一次真正“落地”。
